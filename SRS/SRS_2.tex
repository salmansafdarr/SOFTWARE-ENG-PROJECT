\section{Introduction}

\subsection{Purpose}

\subsubsection{Overview of the Document}
This Software Requirements Specification (SRS) document provides a comprehensive description of the AAK Academy Management System. The document serves as the primary reference for all stakeholders involved in the development, testing, and deployment phases of the project. It establishes a clear understanding of the system's functionalities, constraints, and quality attributes, ensuring alignment between technical teams and the requirement provider.

\subsubsection{Purpose of the SRS}
The purpose of this SRS document is to:
\begin{itemize}
    \item Define all functional and non-functional requirements for the AAK Academy Management System in detail
    \item Serve as a baseline for project planning, design, implementation, and validation activities
    \item Provide clear specifications for quality assurance teams to develop comprehensive test cases
    \item Enable future maintenance teams to understand system architecture and design decisions
    \item Document assumptions, dependencies, and constraints that influence system development
\end{itemize}

\subsection{Scope}

\subsubsection{Product Identification}
AAK Academy Management System is a web-based platform designed to automate academic and administrative operations for AAK Academy, which offers Matric and Intermediate level programs. The system provides a comprehensive solution for student enrollment, subject selection, fee management, assignment distribution, quiz administration, and performance tracking.

\subsubsection{Product Features}

\textbf{What AAK Academy Management System Will Do:}
\begin{itemize}
    \item Provide automated student registration and profile management
    \item Enable subject selection with automatic fee calculation and discount application
    \item Support teachers in creating, distributing, and grading assignments and quizzes
    \item Offer comprehensive administrative controls for managing students, teachers, subjects, and classes
    \item Implement automated discount policies (10\% for 3 subjects, 20\% for 6 subjects)
    \item Provide faculty information pages showcasing teacher profiles and qualifications
    \item Generate academic reports and performance analytics
    \item Support live class integration via Zoom with attendance tracking
    \item Implement secure payment processing for fee collection
    \item Offer role-based dashboards for students, teachers, and administrators
\end{itemize}

\textbf{What AAK Academy Management System Will Not Do (Initial Release):}
\begin{itemize}
   
    \item Include artificial intelligence-based grading or recommendations
    \item Support multiple languages (English only in initial release)
    \item Integrate with external school management systems
    \item Provide real-time video conferencing (uses Zoom integration)
    \item Include parent portals or parental controls
    \item Support offline functionality
    \item Offer social networking features between students
\end{itemize}

\subsubsection{Application of the Software}
AAK Academy Management System addresses critical gaps in academy operations by providing:
\begin{itemize}
    \item \textbf{Operational Efficiency:} Automates manual processes reducing administrative workload
    \item \textbf{Financial Accuracy:} Eliminates errors in fee calculation and discount application
    \item \textbf{Academic Transparency:} Provides clear tracking of assignments, quizzes, and results
    \item \textbf{Accessibility:} Enables 24/7 access to academic materials and information
    \item \textbf{Communication Improvement:} Streamlines interaction between students, teachers, and administrators
    \item \textbf{Data Management:} Centralizes academic and administrative data for better decision making
\end{itemize}

\subsection{Product Perspective}

AAK Academy Management System is an independent web-based platform designed to modernize academic and administrative operations of AAK Academy. Unlike generic school management systems, it is specifically tailored for Matric and Intermediate programs with automated discount calculations and integrated academic tools.

\subsubsection{System Context}

The system operates as a standalone platform with the following contextual relationships:

\textbf{Relationship with Traditional Academy Operations}
AAK Academy Management System serves as a digital transformation tool replacing manual processes:
\begin{itemize}
    \item Replaces paper-based enrollment and registration processes
    \item Automates fee calculation and discount application
    \item Digitizes assignment distribution and submission
    \item Provides online quiz creation and grading
    \item Centralizes academic records and performance data
\end{itemize}

\textbf{Integration Possibilities}
While initially standalone, the architecture allows future integration with:
\begin{itemize}
    \item Payment gateways for online fee collection
    \item Zoom API for live class integration

\section{Specific Requirements}

This section provides detailed specifications for all functional and interface requirements of the AAK Academy Management System. Each requirement is uniquely identified and described with sufficient detail for design and implementation.

\subsection{Functional Requirements}

\subsubsection{User Authentication and Authorization}

\textbf{FR-001: Student Registration}
\begin{itemize}
    \item \textbf{Description:} The system shall allow new students to create accounts through a registration form
    \item \textbf{Inputs:} Full name, father's name, date of birth, gender, contact number, email address, address, previous academic details, program selection (Matric/Intermediate), password, confirm password
    \item \textbf{Process:}
    \begin{enumerate}
        \item Student accesses registration page
        \item Student fills mandatory fields with validation checks
        \item System validates email format, password strength, and age requirements
        \item System creates account in "Pending Verification" status
        \item System sends confirmation email to student
        \item Administrator receives notification of new registration
    \end{enumerate}
    \item \textbf{Outputs:} Account created with pending status, email notifications sent, registration ID generated
    \item \textbf{Validation Rules:}
    \begin{itemize}
        \item Email must be unique in system
        \item Password minimum 8 characters with alphanumeric combination
        \item Contact number must be valid format
        \item Age must be appropriate for selected program
        \item All mandatory fields must be completed
    \end{itemize}
    \item \textbf{Priority:} High
\end{itemize}

\textbf{FR-002: Teacher Registration }
\begin{itemize}
    \item \textbf{Description:} The system shall allow teachers to create faculty accounts
    \item \textbf{Inputs:} Teacher name, qualifications, experience, specialization subjects, contact information, email, temporary password
    \item \textbf{Process:}
    \begin{enumerate}
        \item Teacher accesses faculty management section
        \item Teacher fills faculty registration form
        \item System creates faculty account with "Active" status
        \item System sends account details to teacher's email
        \item Teacher receives credentials for first login
    \end{enumerate}
    \item \textbf{Outputs:} Teacher account created, credentials emailed, subject assignments recorded
    \item \textbf{Priority:} High
\end{itemize}

\textbf{FR-003: User Login}
\begin{itemize}
    \item \textbf{Description:} The system shall authenticate users based on their role (Student/Teacher/Admin)
    \item \textbf{Inputs:} Email/Username, password, user type selection
    \item \textbf{Process:}
    \begin{enumerate}
        \item User enters credentials on appropriate login page
        \item System verifies credentials against database
        \item System checks account status and role
        \item If valid, system creates user session and redirects to role-specific dashboard
        \item If account pending, system displays appropriate message
        \item If credentials invalid, system displays error message
    \end{enumerate}
    \item \textbf{Outputs:} Authenticated session created, appropriate dashboard displayed
    \item \textbf{Security:} Failed login attempts tracked, account locked after 5 consecutive failures
    \item \textbf{Priority:} High
\end{itemize}

\textbf{FR-004: Role-Based Access Control}
\begin{itemize}
    \item \textbf{Description:} The system shall enforce role-based permissions for all system functions
    \item \textbf{Roles and Permissions:}
    \begin{itemize}
        \item \textbf{Student:} Access to personal dashboard, subject selection, assignments, quizzes, grades
        \item \textbf{Teacher:} Access to assigned courses, student management, assessment creation, grading
        \item \textbf{Admin:} Full system access including user management, configuration, reports
    \end{itemize}
    \item \textbf{Process:} System checks user role before granting access to any function or page
    \item \textbf{Priority:} High
\end{itemize}

\textbf{FR-005: Password Management}
\begin{itemize}
    \item \textbf{Description:} The system shall provide comprehensive password management features
    \item \textbf{Features:}
    \begin{itemize}
        \item Password change with current password verification
        \item Password reset via email verification
        \item Password strength enforcement (minimum 8 characters, mixed case, numbers)
     
    \end{itemize}
    \item \textbf{Priority:} Medium
\end{itemize}

\textbf{FR-006: Session Management}
\begin{itemize}
    \item \textbf{Description:} The system shall maintain secure user sessions with appropriate timeouts
    \item \textbf{Parameters:}
    \begin{itemize}
        \item Session timeout after 30 minutes of inactivity for students
        \item Session timeout after 60 minutes of inactivity for teachers
        \item Session timeout after 120 minutes of inactivity for administrators
        \item Warning displayed 5 minutes before timeout
        \item Automatic logout on browser close
        \item Single session per user (prevents multiple concurrent logins)
    \end{itemize}
    \item \textbf{Priority:} Medium
\end{itemize}

\subsubsection{Student Module Requirements}

\textbf{FR-007: Student Dashboard}
\begin{itemize}
    \item \textbf{Description:} The system shall provide personalized dashboard for each student
    \item \textbf{Components:}

        \item Academic calendar view
    \end{itemize}
    \item \textbf{Priority:} High
\end{itemize}

\textbf{FR-008: Subject Selection with Automatic Fee Calculation}
\begin{itemize}
    \item \textbf{Description:} The system shall allow students to select subjects with real-time fee calculation
    \item \textbf{Process:}
    \begin{enumerate}
        \item Student accesses subject selection page
        \item System displays available subjects for selected program
        \item Student selects subjects (minimum 3, maximum 7 for Matric; minimum 3, maximum 6 for Intermediate)
        \item System calculates total fee in real-time
        \item System applies automatic discounts:
        \begin{itemize}
            \item 10\% discount if 3 subjects selected
            \item 20\% discount if 6 subjects selected
        \end{itemize}
        \item Student confirms selection
        \item System generates fee invoice
    \end{enumerate}
    \item \textbf{Outputs:} Subject selection confirmation, fee calculation details, invoice generation
    \item \textbf{Priority:} High
\end{itemize}

    \item \textbf{Features:}

        \item Organized by subject and topic

        \item Progress tracking for completed materials
    \end{itemize}
    \item \textbf{Priority:} High
\end{itemize}

\textbf{FR-010: Assignment Management}
\begin{itemize}
    \item \textbf{Description:} The system shall manage assignment submission and tracking
    \item \textbf{Student Features:}
    \begin{itemize}

        \item Download assignment instructions and resources
        \item Upload completed assignments (multiple file formats supported)
        \item View submission status and confirmation
        \item Receive notifications for upcoming deadlines
        \item View graded assignments with teacher feedback
    \end{itemize}

\subsection{Student Enrollment and Registration System}

\subsubsection{Description and Priority}

\textbf{Priority:} High

1. \textbf{Stimulus:} Prospective student visits academy website and clicks "Apply Now"

2. \textbf{Response:} System displays online registration form with multiple sections

3. \textbf{Stimulus:} Student fills personal information, academic background, and program selection

4. \textbf{Response:} System validates inputs in real-time, shows errors immediately

5. \textbf{Stimulus:} Student submits completed application

6. \textbf{Response:} System creates account in "Pending Verification" status, sends confirmation email

7. \textbf{Stimulus:} Admin reviews application in admin panel

8. \textbf{Response:} System shows complete application with verification checklist

9. \textbf{Stimulus:} Admin approves application

10. \textbf{Response:} System activates account, sends welcome email with login credentials

11. \textbf{Stimulus:} Student logs in for first time

12. \textbf{Response:} System prompts for subject selection with fee calculator

13. \textbf{Stimulus:} Student selects subjects (e.g., 6 subjects)

14. \textbf{Response:} System automatically applies 20\% discount, shows final fee

15. \textbf{Stimulus:} Student confirms selection

16. \textbf{Response:} System generates enrollment confirmation and fee invoice

    \item \textbf{NFR-009:} Database shall support 50 concurrent read/write operations

\subsubsection{Scalability Requirements}
\begin{itemize}

    \item \textbf{NFR-017:} Application architecture shall support load balancing across multiple servers
\end{itemize}

\subsection{Security Requirements}

\subsubsection{Authentication Security}
\begin{itemize}
    \item \textbf{NFR-018:} Passwords shall be stored using bcrypt hashing algorithm with minimum cost factor of 10
    \item \textbf{NFR-019:} Accounts shall be locked after 5 consecutive failed login attempts
    \item \textbf{NFR-020:} Session timeout shall be enforced based on user role and sensitivity
    \item \textbf{NFR-021:} Password policies shall enforce minimum 8 characters with mixed case and numbers
\end{itemize}

\subsubsection{Authorization and Access Control}
\begin{itemize}
    \item \textbf{NFR-023:} Role-based access control shall be implemented for all system functions
    \item \textbf{NFR-024:} Principle of least privilege shall be enforced
    \item \textbf{NFR-025:} Administrative functions shall require re-authentication for sensitive operations
    \item \textbf{NFR-026:} User shall only access their own data unless explicitly granted permission
    \item \textbf{NFR-027:} Audit logs shall track all permission changes and access violations
\end{itemize}

\subsubsection{Data Security}
\begin{itemize}
    \item \textbf{NFR-028:} All sensitive data shall be encrypted at rest using AES-256 encryption
    \item \textbf{NFR-029:} All data transmissions shall use TLS 1.2 or higher
    \item \textbf{NFR-030:} Database connections shall use SSL encryption
    \item \textbf{NFR-031:} File uploads shall be scanned for malware before storage
    \item \textbf{NFR-032:} Backup data shall be encrypted and stored securely
\end{itemize}

\subsubsection{Input Validation and Sanitization}
\begin{itemize}
    \item \textbf{NFR-033:} All user inputs shall be validated both client-side and server-side
    \item \textbf{NFR-034:} SQL injection prevention shall be implemented using parameterized queries
    \item \textbf{NFR-035:} Cross-site scripting (XSS) protection shall be implemented through output encoding
    \item \textbf{NFR-036:} File uploads shall be restricted to approved file types and sizes
    \item \textbf{NFR-037:} CSRF tokens shall be implemented for all state-changing operations
\end{itemize}

\subsubsection{Audit and Logging}
\begin{itemize}
    \item \textbf{NFR-038:} All authentication attempts shall be logged with IP address and timestamp
    \item \textbf{NFR-039:} All data modifications shall be logged with before/after values
    \item \textbf{NFR-040:} Grade changes shall require additional authorization and be fully audited
    \item \textbf{NFR-041:} Financial transactions shall have complete audit trails
    \item \textbf{NFR-042:} Logs shall be retained for minimum 1 year for security analysis
\end{itemize}
