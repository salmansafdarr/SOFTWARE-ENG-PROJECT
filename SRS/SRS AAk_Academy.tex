\section{Introduction}

\subsection{Purpose}
This Software Requirements Specification (SRS) document provides a complete and detailed description of the functional and non-functional requirements of Amar Ahmad Khan Academy (AAK Academy). The purpose of this document is to serve as a reference for developers, designers, project managers, and stakeholders to understand system behavior and constraints.

\subsection{Scope}
AAK Academy is a web-based Learning Management System (LMS) that enables students to enroll in courses, access learning materials, attend live online classes, submit assignments, attempt quizzes, and make online payments. Faculty members can manage course content, assessments, and student performance, while administrators oversee system configuration, reporting, and user management.

\subsection{System Benefits}
\begin{itemize}
    \item Centralized online learning platform
    \item Improved accessibility to educational resources
    \item Efficient academic management
    \item Secure and transparent fee handling
\end{itemize}

\section{Overall Description}

\subsection{Product Perspective}
AAK Academy is a standalone web-based application. It interacts with external services such as a payment gateway and a video conferencing platform. The system follows a client-server architecture and uses a centralized database hosted on cloud infrastructure.

\subsection{Product Functions}
\begin{itemize}
    \item User registration, authentication, and authorization
    \item Course creation and content distribution
    \item Assignment and quiz management
    \item Online live class scheduling and participation
    \item Online payment processing
    \item Performance monitoring and reporting
\end{itemize}

\subsection{User Classes and Characteristics}
\begin{itemize}
    \item Students: Basic computer and internet usage skills
    \item Faculty: Familiarity with teaching tools and assessments
    \item Administrators: Technical and system management expertise
\end{itemize}

\subsection{Operating Environment}
\begin{itemize}
    \item Web browsers (Chrome, Firefox, Edge)
    \item Operating Systems: Windows, macOS, Linux, Android, iOS
    \item Cloud-based hosting environment
\end{itemize}

\subsection{Constraints}
\begin{itemize}
    \item Continuous internet connectivity required
    \item Dependency on third-party APIs
    \item Compliance with data protection laws
\end{itemize}

\subsection{Assumptions and Dependencies}
\begin{itemize}
    \item Users have valid internet access
    \item External services remain available
    \item Institutional policies remain unchanged
\end{itemize}

\section{Functional Requirements}

\subsection{Authentication and Authorization}
\begin{itemize}
    \item FR-001: Users shall be able to register using email and password
    \item FR-002: The system shall authenticate users using JWT
    \item FR-003: The system shall enforce role-based access control
    \item FR-004: Users shall be able to reset passwords
\end{itemize}

\subsection{Student Functions}
\begin{itemize}
    \item FR-005: Students shall access enrolled course materials
    \item FR-006: Students shall submit assignments online
    \item FR-007: Students shall attempt quizzes
    \item FR-008: Students shall view grades and performance reports
    \item FR-009: Students shall make online fee payments
\end{itemize}

\subsection{Faculty Functions}
\begin{itemize}
    \item FR-010: Faculty shall upload learning materials
    \item FR-011: Faculty shall create and evaluate assignments and quizzes
    \item FR-012: Faculty shall schedule and manage live classes
\end{itemize}

\subsection{Administrator Functions}
\begin{itemize}
    \item FR-013: Administrators shall manage users
    \item FR-014: Administrators shall configure courses and fees
    \item FR-015: Administrators shall generate system and academic reports
\end{itemize}

\section{Non-Functional Requirements}

\subsection{Performance Requirements}
\begin{itemize}
    \item NFR-001: The system shall support at least 1000 concurrent users
    \item NFR-002: Page load time shall not exceed 3 seconds
\end{itemize}

\subsection{Security Requirements}
\begin{itemize}
    \item NFR-003: All communication shall use HTTPS
    \item NFR-004: Sensitive data shall be encrypted
    \item NFR-005: Authentication tokens shall be securely managed
\end{itemize}

\subsection{Reliability Requirements}
\begin{itemize}
    \item NFR-006: System uptime shall be at least 99.5\%
    \item NFR-007: Automated daily backups shall be performed
\end{itemize}

\subsection{Usability Requirements}
\begin{itemize}
    \item NFR-008: The interface shall be user-friendly
    \item NFR-009: Minimal training shall be required for users
\end{itemize}

\subsection{Maintainability Requirements}
\begin{itemize}
    \item NFR-010: Modular system design
    \item NFR-011: Proper documentation of code and APIs
\end{itemize}
